# 複数エージェント制作ワークフロー

この文書は、GDM リーディング教材の品質を上げるための役割分担を定める。必要に応じて複数エージェントで並行作業し、最後は統合担当が教材仕様に合わせてまとめる。

## 基本原則

- 最終判断は統合担当が行う。
- 各エージェントは担当範囲を明確にし、他の担当の成果物を勝手に巻き戻さない。
- 生徒用教材には日本語注を入れない。
- 画像は白黒スティックマンだけを使う。
- レッスンは4コマ漫画のように、起承転結と小さなオチを持たせる。
- 印刷用成果物は、A4でそのまま印刷できる HTML を基本にする。

## 推奨フロー

1. 教材設計
2. ストーリー設計
3. 語彙制限チェック
4. GDM文体調整
5. 画像設計
6. 印刷レイアウト
7. 最終レビュー

## 役割

### 教材設計エージェント

役割:
- 対象段階、導入項目、既習項目を整理する。
- そのレッスンで扱う新出項目を少数に絞る。
- 未導入語を避けるための制約を決める。

受け渡し成果物:
- レッスン仕様メモ
- 使ってよい語彙・表現・文型
- 避ける語彙・表現
- 4コマで扱う意味関係

### ストーリー設計エージェント

役割:
- 起承転結のある短い4コマ風ストーリーを作る。
- 小さな問題、意外な変化、短いオチを入れる。
- 絵で意味が取りやすい場面にする。

受け渡し成果物:
- 4コマごとの場面説明
- 英文ドラフト
- オチの説明

### 語彙制限チェックエージェント

役割:
- 本文、タイトル、見出しに未導入語が混ざっていないか確認する。
- 活用形、複数形、所有形、短縮形も確認する。
- 未導入語がある場合は、導入済み語で言い換える。

受け渡し成果物:
- NG語リスト
- 言い換え案
- 許容する例外の有無

### GDM文体調整エージェント

役割:
- 文を短くし、反復を増やす。
- 説明よりも、読んで意味が取れる文に整える。
- 画像と英文が一対一で対応するようにする。

受け渡し成果物:
- 最終英文案
- コマごとの英文
- 必要なら文ごとの導入項目対応メモ

### 画像設計エージェント

役割:
- 各コマを白黒スティックマンだけで表す構図を決める。
- `in/on`, `see/do not see`, `give`, `here/there`, `man/woman` など、理解が難しい関係を絵で支える。
- 画像内テキスト、吹き出し、ラベルを入れない。

受け渡し成果物:
- 4コマ画像の構図メモ
- SVG設計メモ、または画像生成プロンプト
- 人物、物、位置関係の指定

### 画像品質チェックエージェント

役割:
- 画像が白黒のみか確認する。
- 人物がスティックマンに統一されているか確認する。
- 画像内に文字が入っていないか確認する。
- 絵が本文理解を助けているか確認する。

受け渡し成果物:
- 採用可否
- 修正指示
- 分かりにくいコマの再設計案

### 印刷レイアウトエージェント

役割:
- A4でそのまま印刷できる HTML に整える。
- 画像と英文の対応を見やすくする。
- 白黒印刷で線や余白がつぶれないようにする。

受け渡し成果物:
- 完成HTML
- 4コマ配置
- 余白、文字サイズ、改ページ確認メモ

### 最終レビューエージェント

役割:
- `READING_MATERIAL_SPEC.md` との整合を確認する。
- 読み物として面白いか確認する。
- 導入済み語彙、画像、印刷可読性を最終確認する。

受け渡し成果物:
- 修正チェックリスト
- 合格または要修正の判定

## 完成前チェックリスト

- 対象段階に合っているか。
- 導入済み語彙・表現・文型だけで本文が成立しているか。
- 固有名詞以外の未導入語が混ざっていないか。
- 活用形、複数形、所有形、短縮形を確認したか。
- 1コマ目で状況が分かるか。
- 2コマ目で小さな問題や疑問が生まれるか。
- 3コマ目で意外な変化があるか。
- 4コマ目に簡単なオチ、解決、余韻があるか。
- 画像は白黒のみか。
- 人物はスティックマン表現に統一されているか。
- 背景や装飾が複雑すぎないか。
- 画像内に英語・日本語などの文字が入っていないか。
- A4印刷で4コマと本文が切れずに収まるか。
- 文字サイズが中学生向けに読みやすいか。
- 画像と本文の対応が一目で分かるか。
- 白黒印刷でも線や余白がつぶれないか。
- 生徒用ページに日本語訳、語注、文法説明が入っていないか。

## 統合担当の作業

1. 各エージェントの成果物を比較し、矛盾を解消する。
2. 語彙制限を最優先し、その範囲で話の面白さを調整する。
3. 画像が英文理解を助けているか確認する。
4. 印刷用 HTML として最終整形する。
5. 完成前チェックリストに沿って最終確認する。
