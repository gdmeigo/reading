# GDM リーディング教材仕様

## 目的

中学生の英語塾で使うリーディング教材を作る。教材は GDM の段階的導入を前提にし、生徒が「いま読める英語」を少しずつ広げられるようにする。

- NH1: `C:\Users\kazur\reading教材作成\導入項目一覧_NH_2021-2024\NH1_導入項目一覧.docx`
- NH2: `C:\Users\kazur\reading教材作成\導入項目一覧_NH_2021-2024\NH2_導入項目一覧.docx`

参照元の Word ファイルは一覧確認用とし、実際の導入項目は教材フォルダ名に含まれる番号と項目名を優先する。

## 制作体制

- レッスン品質を上げるため、複数エージェントで役割分担して制作・確認してよい。
- 役割分担、受け渡し成果物、完成前レビューは [AGENT_WORKFLOW.md](AGENT_WORKFLOW.md) に従う。
- 最終的な統合担当は、各エージェントの提案をそのまま採用せず、導入順・GDM方針・印刷品質に合うように調整する。

## 導入順の原則

### 2026-07 方針: 単一 Current ID

- 今後の本文生成と検索UIでは、GDM Current ID と NH Current ID を2つ指定しない。
- `Current ID` を1つだけ指定し、そのIDまでに導入済みの語彙・文法を使用可能範囲とする。
- NHとの合流点まではGDM Book1を中心に進める。
- 合流点は `GDM-41-10` とし、そこから先はNHを中心に `NH1`、次に `NH2` へ進める。
- 既存ファイル名に残る `level-N` は内部的な旧コンテンツ整理の名残として残してよいが、教材指定の主軸にはしない。
- 本文量は2段階に整理する。
  - `Short`: 120〜180語程度。短い段落、少数の登場人物、はっきりした起承転結、指定 Current ID の導入項目を1つ以上自然に入れる。
  - `Long`: 300〜450語程度。複数段落、場面・動機・伏線を増やし、指定 Current ID の導入項目を自然に使いながら、未導入構文は避ける。
- サイトのトップページでは、選択した `Current ID` 以下の直近10 IDに紐づくコンテンツ候補を表示する。
- 生成時に特定の `Current ID` が指定された場合、そのIDで導入される語彙・文法項目は本文に必ず自然に入れる。

1. フォルダ名の先頭にある数値プレフィックスを段階番号として扱う。
2. 同じ大番号の中では、枝番号の順に導入する。
3. 番号が同じ、または順序が曖昧な場合は、既存フォルダの並び・参照元一覧・周辺教材の流れを確認する。
4. `まとめ`、`復習`、`Q`、`ワーク` などの項目は、新出項目ではなく定着・確認のための段階として扱う。
5. 導入項目のフォルダ名そのものを、その段階で増やす単語・表現・文型の名前として扱う。

## 系列の扱い

### EP Book1

- 基礎系列として先に進める。
- `1`, `4-1`, `4-2` のような初期項目から、番号順に語彙・代名詞・be 動詞・所有・前置詞・一般動詞などを積み上げる。
- `41-10` 付近までは EP Book1 の導入順を主軸にする。

### NH1

- EP Book1 が `41-10` 付近まで進んだら導入を始める。
- NH1 は中学校教科書系列との接続に使う。
- NH1 側のフォルダ名も、番号順に導入項目として扱う。
- EP で既習の内容は、NH1 では復習・読み替え・表現拡張として扱う。

## `41-10` 付近の運用

EP Book1 の `41-10` 付近は、NH1 へ接続する目安地点とする。

- `41-10` まで: EP Book1 の語彙・文型を中心に、短く確実に読める英文を作る。
- `41-10` 前後: EP の既習項目を使いながら、NH1 の初期項目を少しずつ混ぜる。
- `41-10` 以降: EP の残り項目と NH1 項目を、難度が急に上がらないように並行して扱う。

## 教材作成ルール

- 各教材は、その段階までに導入済みの語彙・表現・文型で読めることを基本にする。
- 新出項目は1教材あたり少数に絞り、既習項目を繰り返し使う。
- 英文は中学生が内容を追いやすい短い場面・会話・説明文を中心にする。
- 科学的内容を必須方針にはしない。主方針は、次の展開が気になって読み進めたくなるストーリーにすること。
- 物語は、見つける、なくす、開ける、逃げる、追う、誤解する、戻ってくる、正体が分かる、などの小さな事件を連鎖させる。
- Example.txt のように、やさしい文を積み重ねながら、箱の中身、逃げたもの、戻ってきた理由などを少しずつ明かす構成を目標にする。
- メインの文法・機能語・動詞は導入順に沿ってグレーディングする。
- 名詞は、ストーリーを成立させるために必要なら導入リストに無くても使ってよい。ただし、一度に増やしすぎず、同じ名詞を反復して意味を定着させる。
- 固有名詞は、日本人が男か女か分かりやすい名前を使う。例: 男の子は Ken, Taro, Ren, Daichi。女の子は Yui, Miki, Hana, Emi, Mika, Saki。
- 登場人物は Ken/Yui のような固定ペアに偏らせない。教材ごとに名前を変え、同じ主人公ばかり続けない。
- 年齢や立場にも幅を持たせる。中学生だけでなく、小さい子、兄姉、保護者、先生、店の人、駅員、高齢者などを自然に混ぜてよい。
- ただし、低レベルでは人物が多すぎると読みにくくなるため、中心人物は2〜3人に絞る。
- 主人公の年齢や関係性が話の骨格に関係するようにする。例: 友だち同士で直す、兄姉が助ける、店の人の困りごとを解く、高齢者を手伝う。
- 話の面白さを重視する。
- 起承転結があるようにする。
- まずは絵なしの文章だけで、`Short` と `Long` の2段階リーディングを作ってよい。
- 本文コンテンツは既存互換のため `level-3.txt`（Short相当）や `level-7.txt`（Long相当）のようなテキストファイルに分離してよいが、UI表示では `Short` / `Long` と呼ぶ。
- HTML は、Current ID と本文量で候補を選び、話ごとの `Short` / `Long` 本文を読み込んで表示する。
- 画像や4コマは必須ではない。必要になった段階で追加する。
- 画像を使う場合は、できるだけ4コマ漫画のように、`起`、`承`、`転`、`結` の流れを作る。
- `起` では人・物・場所の関係をはっきり見せる。
- `承` では小さな問題や誤解を作る。
- `転` では意外な変化、発見、勘違いの発覚などを入れる。
- `結` では短く分かるオチや解決を置く。
- 未導入語を使う必要がある場合は、本文前後で意味が分かるようにするか、新出項目として明示する。
- 文法説明を主役にせず、読んで意味が取れる経験を優先する。
- 日本語注は使ってはいけない。
- 理解が難しそうなところは白黒スティックマンの画像を用いる。参考："C:\Users\kazur\reading教材作成\images"フォルダ内画像

## 本文量設計

- Short: 120〜180語程度。短い段落で、人物・物・場所・小さな変化を明確にする。会話と簡単な過去の出来事を入れてよい。次の場所や次の物が気になる展開にする。Assumed grading には `Short` と Current ID を明示する。
- Long: 300〜450語程度。複数段落で、登場人物、事件、理由、伏線、解決、明るい余韻を持たせる。ジャンルの声を強めてよいが、指定 Current ID より先の未導入構文は避ける。Assumed grading には `Long` と Current ID を明示する。

## 画像ルール

画像を使う場合は、以下のルールに従う。文章だけの教材では、この節は適用しない。

- 教材内の画像は白黒スティックマンだけを使う。
- 色、グレー塗り、写実的な人物、漫画的に細かい服装、装飾背景は使わない。
- 人物は丸い頭と線の体で表し、表情は最小限にする。
- 人物の区別に必要な最小限の特徴は使ってよい。例: Ken は短い髪、Yui はポニーテールと簡単なスカート、teacher は背を高くして眼鏡やネクタイを付ける。
- 人物の特徴は文字ラベルではなく、髪型、背丈、簡単なシルエットで固定する。
- 絵柄は `C:\Users\kazur\reading教材作成\images` 内の既存画像に寄せる。細めの手描き線、余白多め、簡単な机・箱・人物で表す。
- 太いアイコン風、整いすぎた図形、過度な記号表現は避ける。
- 物は単純な線画で表す。例: book, box, pen, water。
- 重要物は、それぞれ判別できる形にする。book は表紙・背表紙・ページ線、pen は二重線の胴体・先端・キャップ、hat はつばと山、box は開いた内側を描く。
- 物の大きさは人物とスケールを合わせる。book は頭より少し大きい程度、pen は前腕程度、hat は頭幅より少し広い程度にする。
- 画像内に日本語、英語ラベル、吹き出し、説明文を入れない。英文は本文側で読ませる。
- 画像は理解が難しい関係だけを助ける。例: `in/on`, `see/do not see`, `give`, `here/there`, `man/woman`。
- スティックマンは、見た目のかわいさより意味の明確さを優先する。
- 同じレッスン内では人物の位置を固定する。例: Ken は左、Yui は右、teacher は中央。
- 視線は点線で示し、誰が何を見ているか分かるようにする。
- 手の向き、体の傾き、物の位置で動作を示す。例: put は手を箱へ、give は手から手へ。
- 物の形を統一する。book は中央線つきの長方形、box は開いた箱、pen は先端つきの細長い棒で描く。
- `do not see` を表すときは、見えない物を描かない。空の箱、視線、見えている別の物で対比を作る。
- オチのコマでは、意外なものを大きめ・中央寄りに描き、本文を読まなくても状況が推測できるようにする。
- 複雑な場面はインライン SVG より PNG を優先してよい。PNG は白背景・黒線のみで、線の太さと人物特徴を安定させる。

## 印刷レイアウト

- 最初の教材形式は、A4 1枚で印刷できる HTML を基本にする。
- 1レッスンはタイトル、導入範囲、4コマ画像、対応する英文で構成する。
- 4コマは上から下へ、または2列×2段で読み順が分かるように配置する。
- 画像と英文は近くに置き、どの英文がどのコマに対応するか一目で分かるようにする。
- 印刷時に色へ依存しない。罫線、余白、文字サイズだけで読めるようにする。
- 日本語注や文法説明欄は入れない。

## 確認観点

- 導入順が数値プレフィックス順になっているか。
- その教材で使った語彙・表現・文型が、導入済み項目の範囲に収まっているか。
- 新出項目が多すぎないか。
- EP から NH1 へ移る地点が `41-10` 付近という方針に沿っているか。
- 中学生向けとして、英文の長さ・語彙量・話題が過度に難しくないか。
- 登場人物の名前、年齢、立場、関係性が既存教材と偏りすぎていないか。
- 4コマとして、起承転結と小さなオチがあるか。
- 画像が白黒スティックマンだけになっているか。
- そのまま印刷したときに、画像と英文が切れたり読みにくくなったりしないか。
